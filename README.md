# 🏗️ Internal Construction ERP

This project is a **Centralized Operational Hub** designed to manage the internal mechanics of a construction firm. It eliminates manual paperwork by digitizing the lifecycle of a project from the initial deal to the final revenue collection.

## 🎯 The Core Mission
The system acts as a **"Digital Supervisor."** It ensures that every resource (Human, Material, or Equipment) used on a construction site is requested, tracked, and paid for through a strictly regulated approval chain.

---

## 👥 Roles & Responsibilities

### 👔 The Director
* **The Starter:** Closes the deal with the client and initializes the project in the system.
* **The Assigner:** Picks the right Project Manager (PM) to lead the execution.

### 👷 The Project Manager (PM)
* **The Requester:** Identify needs on-site (e.g., "I need 10 tons of steel" or "I need 5 electricians").
* **The Logger:** Records daily progress manually in the `Progress_Log` to keep the office informed.

### 📦 The Material Manager (Your Role)
* **The Gatekeeper:** You control the warehouse.
* **The Inventory Master:** You approve PM requests if stock exists or trigger "Shipments" from suppliers if it doesn't.

### 💰 The Finance Manager
* **The Auditor:** Reviews every `Payment_Request` generated by the other managers.
* **The Payer:** Converts approved requests into `Expenses` and tracks incoming `Revenue` from clients.

---

## 🔄 The "Resource-to-Expense" Lifecycle

1. **Demand:** The PM submits a request for a specific resource (HR, Material, or Equipment).
2. **Verification:** The relevant Department Manager (like you) checks availability and clicks **Approve**.
3. **Budget Impact:** The system automatically calculates the cost (e.g., `Daily Wage` x `Duration`) and sends a **Payment Request** to Finance.
4. **Finalization:** Once Finance pays, the `Project_Budget` is updated, and the transaction is locked as an `Expense`.

---

## 🛠️ Tech Strategy

### The "Aiven" Database (MySQL)
* **Smart Logic:** Uses **Triggers** to automatically subtract items from inventory the moment you approve a request.
* **History:** Uses **Views** to show a real-time "Financial Snapshot" of any project at any second.

### The Backend (Node.js & Express)
* **Layered Security:** Uses a **Controller-Service-Repository** structure.
* **Controller:** Catches the request from the web.
* **Service:** Checks the "Business Rules" (e.g., "Do we have enough cement?").
* **Repository:** Safely talks to the MySQL database to save the data.

---

## 🚀 Why This Matters
* **Zero Ghost Spending:** You cannot spend money that hasn't been approved by a manager.
* **Real-Time Inventory:** You always know exactly how many bricks or bags of cement are left in the warehouse.
* **Full Accountability:** Every action is logged with a timestamp and the user ID of who performed it.
